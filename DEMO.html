Introduction

The assessment brief for the CMP1130M Assessment item 1 was to design an interactive website for a band or brand and examine the strengths and weaknesses of the current web specifications and standards which we were required to use while developing the website. Therefore, the website has been developed using HTML5 for the content and CSS3 for the styling of the content. The latest versions of jQuery and JavaScript were used to increase the functionality of the website. The standards were used to feature hyperlinks, text effects, multimedia, local web storage, geolocation and advanced JavaScript functionality. To ensure quality of the website we collected user feedback during the design process of wireframing, where subsequent feedback was turned into changes or improvements.

Web Standards and Validation

Using the latest web standards means development is easier as there is more support from experienced users to receive help from and makes maintenance easier as interoperability is stronger. This also allows maintenance and running costs to be lower. Using a range of web specifications and standards, splits up the content customisation allowing for more control over the functionality of the website. HTML5 requires less maintenance and is quick to code as it is one code base for all operating systems, making it a cost-effective solution for developing a site for multiple platforms (PC, tablet, phone etc…). It is also very widely used so support online is in large supply and learning it will become a very useful skill. However, it can be difficult to link the code to the other standards if you are inexperienced, on the other hand, this is mainly a fault with the other standards as html is the main code. CSS3 makes it easy to control how the HTML looks but requires a lot of lines and is very different to HTML and causes the downloading of HTML pages to increase if CSS is embedded, again, it is widely used. However, certain CSS code requires files to be installed on the user’s computer such as font. JavaScript is a simple programming language but can be annoying when programming as error messages are quite vague and unhelpful. JS can be used to massively increase the functionality of your website. jQuery is a helpful tool to increases functionality of JavaScript, to create and use local storage, but as it uses API’s, error messages are very vague and can be hard to debug, there is support online, but less than other standards like CSS. Overall, using a range of web standards requires coding to be stricter during development so that the standards can work together. This may have an impact on the result of the website as stricter coding requires a higher programming knowledge and is harder to achieve validation from W3C. Commenting has also been used so any other programmer could either learn from the code or edit it.
